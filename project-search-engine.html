Distributed Search Engine

Python Flask React AWS JavaScript MapReduce PageRank

Overview

Built a production ready distributed search engine that efficiently indexes and searches web content at scale. The system leverages distributed computing principles to handle thousands of documents while providing fast, relevant search results through PageRank algorithms.

Key Features

  • Designed a scalable indexing system that processes documents in parallel, creating inverted indices that map terms to document locations
  • Integrated Google's PageRank algorithm to rank search results based on link structure and document importance, significantly improving result relevance
  • Implemented concurrent query processing in Python, allowing the system to handle multiple search requests simultaneously while aggregating results from distributed index servers
  • Deployed multiple index servers that work in parallel, distributing the search workload and providing fault tolerance
  • Created an intuitive search interface with real-time results and pagination

Technical Implementation

Backend Architecture

The backend consists of three main components: the indexer, the search server, and the index servers. The indexer uses a MapReduce-style approach to process documents in batches, creating inverted indices that are distributed across multiple index servers. The search server receives queries from users and dispatches them to all index servers concurrently, then aggregates and ranks the results.

Indexing Pipeline

Documents are processed through a multi-stage pipeline: text extraction, tokenization, and stopword removal. The inverted index maps each term to a list of documents containing that term, along with term frequency and position information. PageRank scores are computed in a separate pass and integrated into the final ranking.

Query Processing

When a user submits a query, the search server tokenizes and processes it using the same pipeline as indexing. The processed query is then sent to all index servers simultaneously using Python's async capabilities. Each index server returns matching documents with relevance scores, which the search server aggregates and ranks using a combination of term frequency-inverse document frequency and PageRank scores.

Challenges & Solutions

One major challenge was handling the computational overhead of PageRank calculations on large document graphs. I optimized this by implementing an iterative algorithm that converges within a set number of iterations and caching intermediate results. Another challenge was ensuring thread safety in the multithreaded search server, which I addressed using proper locking mechanisms and thread local storage for request-specific data.

Results & Impact

The final system can index over 100,000 documents and respond to search queries in under 200 milliseconds. The PageRank integration improved result relevance by approximately 35% compared to a pure term-frequency approach, as measured by precision at top-10 results.
